# Использование прокси

В разных частях плагина вы можете найти возможность установить прокси для доступа к какому-то сервису. Например, так вы можете обращаться к ВК, если в стране, где находится ваш сервер, он заблокирован.

В первую очередь вам нужно понять ограничения:

::: danger
Возможность использования прокси в JVM довольно ограниченная. 

Вы НЕ можете использовать "крутые" протоколы по типу vless. Если государство хоть немного старается реально заблокировать сервис, HTTP / SOCKS прокси __сами по себе__ НЕ смогут обойти блокировку.

Вы НЕ можете настраивать аутентификацию для SOCKS прокси прямо в конфиге - нужно менять параметры JVM, например, через аргументы запуска.

В первую очередь эти настройки существуют для того, чтобы вы могли перенаправить запросы на свой прокси, который отправит их так, как вам нужно.
:::

### Протокол HTTP

Пример настройки:

```yml
enabled: true
protocol: !<http>
  url: 'http://example.com:8080'
extra-headers:
  'Proxy-Authorization': 'Basic YWxhZGRpbjpvcGVuc2VzYW1l'
```

Здесь наш прокси это `example.com:8080`, а авторизуемся мы через заголовок [`Proxy-Authorization`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Proxy-Authorization).

### Протокол SOCKS

Пример настройки:

```yml
enabled: true
protocol: !<socks>
  host: '127.0.0.1'
  port: 9050
extra-headers: {}
```

Здесь мы обращаемся к локальному прокси-серверу на порту 9050. 
Этот пример можно использовать, чтобы пропускать трафик через Tor - при установке на Linux сервер он поднимает SOCKS сервер на этом порту.
