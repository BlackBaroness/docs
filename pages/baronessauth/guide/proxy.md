# Использование прокси

В разных частях плагина вы можете найти возможность установить прокси для доступа к какому-то сервису. 
Например, так вы можете обращаться к Telegram, если в стране, где находится ваш сервер, он заблокирован.

В первую очередь вам нужно понять ограничения:

::: danger
Вы можете использовать только HTTP или SOCKS прокси.

Если вы хотите использовать, например, VLESS, вам нужно установить,
например, xray, чтобы тот развернул локальный SOCKS прокси.

Простых обходов блокировок по типу "включить параметр в конфиге плагина" нет, потому что это требовало бы, чтобы
я давал всем доступ к своим личным прокси, а это большая ответственность.
Возможность использования прокси в JVM довольно ограниченная.
:::

### Протокол HTTP

Пример настройки:

```yml
enabled: true
protocol: !<http>
  url: 'http://example.com:8080'
extra-headers:
  'Proxy-Authorization': 'Basic YWxhZGRpbjpvcGVuc2VzYW1l'
```

Здесь наш прокси это `example.com:8080`, а авторизуемся мы через заголовок [`Proxy-Authorization`](https://developer.mozilla.org/en-US/docs/Web/HTTP/Reference/Headers/Proxy-Authorization).

### Протокол SOCKS

Пример настройки:

```yml
enabled: true
protocol: !<socks>
  host: '127.0.0.1'
  port: 9050
extra-headers: {}
```

Здесь мы обращаемся к локальному прокси-серверу на порту 9050. 
Этот пример можно использовать, чтобы пропускать трафик через Tor - при установке на Linux сервер он поднимает SOCKS сервер на этом порту.
